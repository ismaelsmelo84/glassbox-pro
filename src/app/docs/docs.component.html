GlassBox é uma ferramenta inovadora, que foi desenvolvida com exclusividade pela OnSizing. Ela foi fundamentada nos principais padrões internacionais de estimativa e medição atualmente existentes, os quais derivam da ISO/IEC 14.143-1.

Com GlassBox, a operação de software (e seus projetos) passa a ser avaliada através de um conjunto consistene de KPIs, conforme as necessidades de negócio e de uso consciente dos recursos. Esta abordagem viabiliza a tomada de decisão nos diferentes níveis, assim como a prova das ações de melhoria, afirmando-se, desta forma, como modelo de trabalho transparente e sustentável.

O conjunto de medidas obtidas é passível de comparação com o mercado. Clique aqui para mais detalhes.
São disponibilizados dados relacionados às perspectivas de "Diagnóstico", "Progresso" e "Desempenho" da operação, os quais são publicados em diferentes níveis de abstração da informação.

Em relação ao "Diagnóstico", são exemplos de informações:

  • Capacidade - avaliação de quanto a operação consegue produzir;
  • Complexidade - avaliação objetiva da complexidade dos ambientes (tecnológicos e de negócio);
  • Saúde do sistema - acompanhamento da evolução do trabalho a fazer, fazendo e feito.

A respeito do "Progresso", são disponibilizados dados relacionados principalmente aos projetos e épicos da organização, tais como:

  • Realizado - trabalho realizado ou efetivo em relação aos compromissos assumidos;
  • Completude - percentual construído em relação às expectativas iniciais;
  • Escala de evolução - razão entre o realizado e a expectativa inicial.

Por sua vez, estará disponível também um modelo de acompanhamento dos times como "Plano de Desenvolvimento Coletivo", que compreende os seguintes domínios:

  • Eficácia - construímos a quantidade de software conforme nos comprometemos, com a qualidade adequada;
  • Eficiência - otimizamos o uso dos recursos para o desenvolvimento da solução de software;
  • Efetividade - geramos os resultados esperados;
  • Maturidade - estamos bem estruturados;
  • Clima - temos um bom clima para trabalhar.

This template is built with angular-cli and requires Node 6.9.0+ and NPM 3+ to be installed.

Installing the prerequisites

Node.js and NPM: You can download Node.js from https://nodejs.org. NPM comes bundled with Node.js

angular-cli: You can install angular-cli by executing from your terminal npm install -g @angular/cli. More details can be found here https://github.com/angular/angular-cli

Installing the npm packages

Before proceding you'll need to install npm packages. You can do this by running npm install from the root of your project to install all the necessary dependencies.

Development server

Run ng serve for a dev server. Navigate to http://localhost:4200/. The app will automatically reload if you change any of the source files.

Code scaffolding

Run ng generate component component-name to generate a new component. You can also use ng generate directive/pipe/service/class.

Building

Run ng build to build the project. The build artifacts will be stored in the dist/ directory. Use the --prod flag for a production build. Use --aot flag for a AoT build

Further help

To get more help on the angular-cli use ng --help or go check out https://github.com/angular/angular-cli.
